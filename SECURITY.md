## Reporting a Vulnerability

The security of our project is taken very seriously. We appreciate your efforts to responsibly disclose your findings and will make every effort to acknowledge your contributions.

### How to Report

To report a security vulnerability, please **DO NOT** use public GitHub Issues.

Instead, send an email to: **contato@elevafin.com.br**

Include the following information:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source code files related to the manifestation of the issue
- Location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

### What to Expect

- **Acknowledgment**: You will receive a response confirming receipt within 48 hours
- **Assessment**: We will evaluate the issue and determine its severity and impact
- **Resolution**: We will work on a fix and coordinate disclosure with you
- **Credit**: With your permission, we will give credit for the discovery

### Disclosure Policy

- Please give us reasonable time to fix the issue before public disclosure
- Do not access or modify other users' data
- Do not perform actions that could negatively affect the service

## Security Practices

We are committed to:

- Keeping all dependencies up to date
- Regularly reviewing code for vulnerabilities
- Following security best practices in development
- Responding quickly to security reports

## Security Updates

Security updates are disclosed through:

- Repository commits tagged with `[SECURITY]`
- GitHub Security Advisories
- Release notes

Thank you for helping keep ElevaFin and our community safe!
